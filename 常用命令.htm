连接命令

  • quit;或exit;——断开数据库连接并退出sqlplus环境
  • disc;——断开数据库连接但不退出sqlplus
  • conn 用户名/密码; ——连接用户（管理员用户sys连接的时候需要在后面添加as sysdba或者as sysoper）

交互式命令

  • show user;——显示当前登录用户
  • show errors;——显示代码中的错误
  • clear screen;——清除屏幕
  • edit 路径;——编辑指定路径的sql脚本
  • spool 路径;—>sql语句运行结果—>spool off; ——可以将两个spool之间的内容输出到指定路径的文件中
  • alter user 用户名 account unlock;——解锁用户（解锁用户必须是管理员用户）
  • create user 用户名 identified by 密码;——创建用户命令（创建用户必须是管理员用户，密码不能以数字开头）
  • alter user 用户名 identified by 密码;——修改用户密码命令（修改用户密码既可以是管理员用户，也可以是本用户）
  • grant connect to 用户名;——授予用户连接权限（新建的用户是没有任何权限的，必须通过管理员用户授权才能有权限）
  • grant resource to 用户名;——授予用户控制资源权限（oracle对象、表、簇、序列、过程、视图、plsql块、触发器）的权限
  • grant all on 方案名.数据库对象 to 用户名;——分享权限（如果想把得到的权限再分享出去就需要一开始授权的时候在后面加上with grant option）
  • revoke all on 方案名.数据对象 from 用户名;——回收用户权限
  • drop user 用户名;——删除用户（删除用户必须是管理员用户，删除有对象的用户就要在最后加上cascade）
  • select * from emp where job='&job';——表示查出emp表中job字段为用户输入值的所有记录
    select——在sql语句中表示的是查询
    *——表示所有的记录
    emp——表示查询的是emp表
    where——根据什么条件来查询
    &——表示可以替代变量，而该变量在执行时，需要用户输入 set serverout on;——打开服务器端输出 ，dbms_output.put_line()的输出内容可以显示

表命令

  • desc 表名;——显示表结构
  • column 列名 fornat 格式;——设置一行显示的字符数
  • set pagesize 行数;——设置一页显示的行数
  • set pagesize 行数;——设置一页显示的行数
  • create table 表名(字段1 数据类型1, 字段2 数据类型2……);——创建表
  • alter table 表名 add(要添加的字段 数据类型);——添加列
  • alter table 表名 modify(需要修改的列 数据类型); ——修改列
  • alter table 表名 drop(需要删除的字段);——删除列
  • rename 原表名 to 新表名;——修改表名
  • insert into 表名(要插入的字段, ……) values(字段的值, ……);——将数据插入到表指定字段中
  • insert into 表名 values(字段1的值, 字段2的值);——将数据插入到表全部字段中
  • update 表名 set 需要修改的字段='新的值' where 条件;——修改表中的数据
  • delete 表名;——删除表中的所有记录，但是保留表结构
  • drop table 表名;——删除表中的所有记录和表结构
  • delete 表名 where 条件;——删除表中满足条件的数据
  • truncate table 表名;——删除表中的所有记录，表结构还在，不写日志，无法找回删除的记录，速度快
